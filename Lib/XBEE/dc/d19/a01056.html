Detailed Description

Base class for Xbee application implementations.

The ApplicationBase class provides functionalities for managing Xbee module connections, handling incoming and outgoing messages, and managing various Xbee services such as spectrum analysis, network discovery, and channel scanning.

◆ start()

virtual bool Lib::Xbee::API::ApplicationBase::start ( bool pause)
inlineprotectedvirtual

Starts the application and initializes the Xbee module.

This function performs the following steps:

  1. Asserts that the application is not already initialized.
  2. Adds services to the service manager.
  3. Initializes the Xbee module and sets up necessary callbacks.
  4. Waits for the hardware reset callback to confirm module presence.
  5. Creates message queues for received messages and transmit delivery statuses.
  6. Reads parameters from the module.
  7. Starts the application task.
  8. Optionally pauses the application if the pause flag is set.
Parameters
pauseIf true, the application will be paused after starting.
Returns
true if the application started successfully, false otherwise.

Reimplemented in Lib::Xbee::API::ClientBase, and Lib::Xbee::API::ServerBase.

◆ sendData()

virtual bool Lib::Xbee::API::ApplicationBase::sendData ( const std::vector< uint8_t > & vector)
inlineprotectedvirtual

Sends data using the Xbee module.

This function sends data encapsulated in a frame using the Xbee module. It first checks if the module is present. If the module is not present, it returns false. Otherwise, it constructs a frame with the given data and attempts to send it. If the frame is sent successfully, it increments the count of sent data messages and returns true. If the frame sending times out, it increments the count of timed-out data messages and returns false.

Parameters
vectorA reference to a vector containing the data to be sent.
Returns
true if the data was sent successfully, false otherwise.

Lib::Xbee::Core::RemoteAtCommand

Detailed Description

Provides methods to send and receive remote AT commands to and from an XBee module.

The RemoteAtCommand class provides methods to open and close the service, send commands, and receive responses. It uses FreeRTOS queues to manage message passing and ensures proper synchronization and error handling.

void Lib::Xbee::Core::RemoteAtCommand::open ( )
inline

Opens the service and sets up the necessary event handlers.

This function checks if the message queue is already created. If it is, it asserts an error and returns. Otherwise, it sets up the event handler for remote AT command responses and creates the message queue.

Note
The function will assert if the service is already open or if the message queue creation fails.

◆ close()

void Lib::Xbee::Core::RemoteAtCommand::close ( )

Closes the Remote AT Command interface.

This function checks if the message queue is created. If it is, it sets the onRemoteATCommand event handler to nullptr and releases the message queue.

Lib::Xbee::Core::Frame0x97

Detailed Description

Represents a response frame with ID 0x97 from an XBee device.

Remote AT Command Response- 0x97

This frame type is emitted in response to a Remote AT Command Request - 0x17. Some commands send back multiple response frames; for example, the ND command. Refer to individual AT command descriptions for details on API response behavior. This frame is only emitted if the Frame ID in the request is non-zero.

This file contains the definition of the Frame0x97 class, which is used to parse and store the data received in a response frame with ID 0x97 (Remote AT Command Response) from an XBee device. The class provides functionality to extract and manage various fields such as frame ID, source address, AT command response, status, and additional parameters.

Note
The 16-bit destination address field is typically set to 0xFFFE and is unused in this context.
Example Usage:
Frame::Data frameData = ...; // Obtain frame data from the device
Frame0x97 responseFrame(frameData); // Parse the frame data
uint8_t frameId = responseFrame.frameId; // Get the frame ID
uint64_t dstAddr64 = responseFrame.dstAddr64; // Get the 64-bit destination address
uint16_t dstAddr16 = responseFrame.dstAddr16; // Get the 16-bit destination address
uint16_t atCmd = responseFrame.atCmd; // Get the AT command response
uint8_t status = responseFrame.status; // Get the status
std::vector<uint8_t> parameters = responseFrame.parameters; // Get the parameters

This vector holds the parameters in the form of 8-bit unsigned integers (uint8_t). It is used to manage and manipulate a collection of byte-sized data. uint8_t Lib::Xbee::Core::Frame0x97::status

Status of the response frame.

This variable holds the status information of the response frame. The status can indicate various states or results of the frame processing.

Status code for the host's request:
  • 0x00 = OK
  • 0x01 = ERROR
  • 0x02 = Invalid command
  • 0x03 = Invalid parameter
  • 0x04 = Transmission failure
  • 0x0C = Encryption error

Lib::Xbee::Core::Frame0x00

Detailed Description

Represents a 64-bit Transmit Request frame (Frame 0x00) for XBee communication.

This class is used to construct and manage the data for a 64-bit Transmit Request frame, which is used to send data to a specific 64-bit address in an XBee network.

The Frame0x00 class inherits from QueryBase and provides a type conversion operator to convert the frame data into a Frame::Data object. It includes the following fields:

  • frameId: Identifies the data frame for the host to correlate with a subsequent response. If set to 0, the device will not emit a response frame.
  • dstAddress: The 64-bit IEEE address of the destination device. If set to 0x000000000000FFFF, the broadcast address is used.
  • options: A bit field of options that affect the outgoing transmission. The options include:
    • Bit 0: Disable MAC ACK [0x01]
    • Bit 2: Send packet with Broadcast PAN ID [0x04] (802.15.4 firmwares only) Note: Option values may be combined. Set all unused bits to 0.
  • rfData: The serial data to be sent to the destination. Use NP to query the maximum payload size that can be supported based on current settings.
Note
This class is specific to the XBee communication protocol and is used within the context of the XBee library.

Frame ID.

Identifier for the frame.

Identifies the data frame for the host to correlate with a subsequent response. If set to 0, the device will not emit a response frame. The frame ID is used to correlate the response frame with the original data frame. The frame ID is a value between 1 and 255. If set to 0, the device will not emit a response frame.

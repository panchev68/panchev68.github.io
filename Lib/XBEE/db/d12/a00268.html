Lib::Xbee::API::Connection::Info::Period

Detailed Description

A class to measure and track time periods with minimum, maximum, and average values.

The Period class provides functionality to measure time intervals, track the minimum and maximum values, and calculate a smoothed average value. It uses FreeRTOS tick counter for time measurement.

Begins the measurement process.

This function starts the measurement process by setting the _isStarted flag to true and recording the current tick count using FreeRTOS::TickCounter::get(). If the measurement process has already been started (_isStarted is true), the function returns immediately without performing any action.

◆ endMeasurement()

void Lib::Xbee::API::Connection::Info::Period::endMeasurement ( )
inline

Ends the measurement process and updates the measurement statistics.

This function stops the measurement process if it was started, calculates the elapsed time since the measurement started, and updates the measurement value, raw value, minimum, and maximum values accordingly.

The function performs the following steps:

  • Checks if the measurement process was started. If not, it returns immediately.
  • Sets the measurement process as stopped.
  • Calculates the elapsed time since the measurement started.
  • If the current measurement value is zero, it sets it to the elapsed time.
  • If the absolute difference between the current elapsed time and the raw value is greater than or equal to 5, it updates the measurement value to the elapsed time. Otherwise, it averages the current measurement value and the elapsed time.
  • Updates the raw value to the elapsed time.
  • Updates the minimum value if the elapsed time is less than the current minimum or if the minimum is zero.
  • Updates the maximum value if the elapsed time is greater than the current maximum or if the maximum is zero.

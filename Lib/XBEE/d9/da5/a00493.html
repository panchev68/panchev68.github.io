Lib::Xbee::Core::Frame0x81

Detailed Description

Represents a Frame-0x81 object used for handling specific frame data.

The Frame0x81 class is responsible for extracting and converting the source address, RSSI, and option fields from the provided frame data. It also stores the remaining payload data in the rfData member.

This class is designed to handle frames of type 0x81, which include information such as the sender's 16-bit network address, the received signal strength indicator (RSSI), and various options related to the frame's behavior or properties.

Example Usage:
  • Frame::Data frameData = ...; // Obtain frame data from some source
    Frame0x81 frame(frameData); // Parse the frame data
    uint16_t sourceAddress = frame.srcAddr16; // Get the source address
    uint8_t signalStrength = frame.rssi; // Get the RSSI value
    uint8_t frameOption = frame.option; // Get the options
    std::vector<uint8_t> payload = frame.rfData; // Get the RF data

uint8_t Lib::Xbee::Core::Frame0x81::rssi = 0

Received Signal Strength Indicator (RSSI) value.

This variable holds the RSSI value, which indicates the strength of the received signal. The value is represented as an unsigned 8-bit integer.

The Hexadecimal equivalent of (- dBm) value. For example if RX signal strength is -40 dBm, then 0x28 (40 decimal) is returned.

◆ srcAddr16

uint16_t Lib::Xbee::Core::Frame0x81::srcAddr16 = 0

16-bit source address.

This variable holds the 16-bit address of the source device. The 16-bit network address where the packet was delivered (if successful). If not successful, this address is 0xFFFD (destination address unknown). 0xFFFE indicates 16-bit addressing was not used.

The sender's 16-bit network address.

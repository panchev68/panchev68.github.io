RTOS::Task Class Reference

Detailed Description

A C++ wrapper for FreeRTOS threads.

This class provides a C++ interface for creating and managing FreeRTOS threads.

Template Parameters
StackSizeThe stack size of the thread. Default is configMINIMAL_STACK_SIZE.
PriorityThe priority of the thread. Default is 2.

enum class RTOS::Task::NotifyAction
strong

Enumeration of possible notification actions.

This enumeration defines the possible actions that can be taken when sending a notification to a thread. The actions include setting bits, incrementing a value, and setting a value with or without overwriting.

Enumerator
NO_ACTION 

No action.

SET_BITS 

Set bits.

INCREMENT 

INCREMENT value.

SET_VALUE_WITH_OVERWRITE 

Set value with overwrite.

SET_VALUE_WITHOUT_OVERWRITE 

Set value without overwrite.

Creates and starts a FreeRTOS task with the specified function and arguments.

This function creates a new FreeRTOS task that executes the provided callable object with the given arguments. The task is created with the specified stack size and priority.

Template Parameters
_CallableThe type of the callable object.
_ArgsThe types of the arguments to be passed to the callable object.
Parameters
functionThe callable object to be executed by the task.
argsThe arguments to be passed to the callable object.
Returns
true if the task was successfully created and started, false otherwise.
Note
The task function is responsible for deleting the callable object and deleting itself (vTaskDelete) when it completes execution.

◆ yield()

static void RTOS::Task::yield ( )
inlinestaticnoexcept

Yields the processor to another task.

This function is a wrapper around the FreeRTOS taskYIELD() macro, which forces a context switch to another task that is ready to run. It can be used to voluntarily relinquish the CPU, allowing other tasks of equal or higher priority to execute.

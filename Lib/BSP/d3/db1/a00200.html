inline

Task function template specialization for parallel update mode.

This function performs parallel updates of inputs and outputs. It is possible when the SPI port is different for both interfaces. The function runs in an infinite loop, performing the following steps in each iteration:

  1. Get the current tick count.
  2. Handle pre-update event.
  3. Call the pre-update callback for inputs.
  4. Update inputs.
  5. Call the pre-update callback for outputs.
  6. Update outputs.
  7. Handle post-update event.
  8. Trigger the tick event if it is set.
  9. Delay the task for 1 millisecond.

Task function template specialization for sequential update mode.

This function performs sequential updates of inputs and outputs. It is necessary when the SPI port is common to both interfaces and reinitialization is required when servicing each device. The function runs in an infinite loop, performing the following steps in each iteration:

  1. Get the current tick count.
  2. Handle pre-update event.
  3. Call the pre-update callback for inputs.
  4. Update inputs.
  5. Call the pre-update callback for outputs.
  6. Update outputs.
  7. Handle post-update event.
  8. Trigger the tick event if it is set.
  9. Delay the task for 1 millisecond.

Detailed Description

TouchGFX Display Driver Base Class.

BSP::Display::DriverBase class

Базов клас за дисплени дайвери съвместим с TouchGFX


Създаване

Има 3 тъпа драйвери :

  1. Когато се използва Частичен Буфер Този тип е поднодящ за 16-bits TFT SPI Driver. Тогава фунцкцията update () не се използва. предимства : Използване на много малко RAM Памет.

  2. Когато се използва Единичен Буфер

    Поради факта че TouchGFXпрехвърля само блокове (области от дислеят) които са променети . При създаване на драйвер за B/W Дисплей оравим следното:

    Конфигурираме в STM32QubeMX/TouchGFX по следният начин

    Така се създава единичен буфер в който TouchGFX рендва екрана в паметта.

    адресът на този буфер може да се вземе с TouchGFXGeneratedHAL::getInstance ()->getTFTFrameBuffer () функцията.

    TouchGFX Transferring the framebuffer

    за да може да се свърже с TouchGFX трябва да се добави следният ред

    **file : TouchGFX\target\TouchGFXHAL.cpp**

    in file file : TouchGFX\target\TouchGFXHAL.cpp
    void TouchGFXHAL::flushFrameBuffer(const touchgfx::Rect& rect)
    {
    // update entry display
    BSP::OLED.writeWindow (TouchGFXGeneratedHAL::getInstance ()->getTFTFrameBuffer(), rect.x, rect.y, rect.width, rect.height );
    TouchGFXGeneratedHAL::flushFrameBuffer (rect);
    }

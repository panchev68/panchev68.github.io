virtual void Lib::BSP::DigitalPeripheralBase::handlePostUpdateEvent ( )
inlineprotectedvirtual

Data processing after the update.

This function is called after the peripheral update and serves to synchronize the data received from the peripheral and the DAL::DigitalPeripheralData. It synchronizes peripheral inputs and peripheral output status with DAL::DigitalPeripheralData.

Postcondition
Must be implemented in the owner class

Data processing before the update.

This callback is called before the peripheral update and serves to synchronize the data that needs to be sent to the peripheral. It synchronizes the peripheral outputs with DAL::DigitalPeripheralData.

Postcondition

Processing outputs in test mode.

This function is used to synchronize the output port values ​​in test mode. Usually I need to change the output values ​​in DAL::DigitalPeripheralData so that they can be displayed on the display. This function is called at a certain time after the test output value is formed. see : DigitalPeripheralBase::Test::handleTickEvent

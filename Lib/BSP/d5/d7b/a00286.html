Lib::BSP::DigitalPeripheralBase::Outputs Class Reference

Detailed Description

DigitalPeripheralBase::Outputs class.

This class provides an interface for handling output data for the digital peripheral. It includes functions for setting up the outputs, updating the outputs, and getting the output value. The class also includes events for when the output value or status changes.

◆ update()

void Lib::BSP::DigitalPeripheralBase::Outputs::update ( )
inlineoverridevirtual

Updates the state of the digital peripheral.

This function performs the following tasks:

  1. Checks if there is a value change and processes the output data accordingly.
  2. Updates the timeout timer and checks if it has expired.
  3. If there is an update, it sends and receives data to/from the SPI port.
  4. Enables outputs on the first transaction.
  5. Detects SPI errors and invokes the onSpiError callback if an error occurs.
  6. Processes the status of the MC33879 and updates the status if it has changed.

The function uses the following member variables:

  • _isValueChanged: Indicates if there is a value change.
  • _bitset: Stores the current value of the pins.
  • _txBuffer: Buffer for transmitting data.
  • _rxBuffer: Buffer for receiving data.
  • _numberOfPins: Number of pins to process.
  • _mapFields: Mapping of pins to their respective ports and masks.
  • _timeout: Timer for status updates.
  • _changesCounter: Counter for the number of changes.
  • _diagCounter: Counter for diagnostics.
  • _nss: SPI select pin.
  • _spi: SPI interface.
  • _oe: Output enable pin.
  • _isSpiError: Indicates if there is an SPI error.
  • onSpiError: Callback function for SPI errors.
  • onStatusChanged: Callback function for status changes.
  • _status: Current status of the pins.

Implements Lib::BSP::DigitalPeripheralBase::PortBase.

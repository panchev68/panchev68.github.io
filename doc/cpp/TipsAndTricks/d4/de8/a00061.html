Command Design Pattern

Introduction

The Command Design Pattern is a behavioral design pattern that turns a request into a stand-alone object that contains all information about the request. This transformation allows for parameterization of methods with different requests, queuing of requests, and logging of the requests. It also provides support for undoable operations.

Example

Here is an example implementation of the Command Design Pattern in C++:

// Command interface
class Command {
public:
virtual ~Command() {}
virtual void execute() = 0;
};
// Concrete Command
class ConcreteCommand : public Command {
private:
Receiver* receiver;
public:
ConcreteCommand(Receiver* r) : receiver(r) {}
void execute() override {
receiver->action();
}
};
// Receiver
class Receiver {
public:
void action() {
std::cout << "Receiver action executed" << std::endl;
}
};
// Invoker
class Invoker {
private:
Command* command;
public:
void setCommand(Command* cmd) {
command = cmd;
}
void executeCommand() {
command->execute();
}
};
int main() {
Receiver receiver;
ConcreteCommand command(&receiver);
Invoker invoker;
invoker.setCommand(&command);
invoker.executeCommand();
return 0;
}

In this example, the Command interface declares a method for executing a command. The ConcreteCommand class implements this interface and defines the binding between a Receiver object and an action. The Invoker class is responsible for initiating the request by calling the execute method on the command.

The Receiver class contains the actual business logic that needs to be executed when the command is called. The Invoker class holds a command and at some point asks the command to carry out a request by calling its execute method.

This pattern decouples the classes that invoke the operation from the classes that perform the operation. It allows for the creation of a sequence of commands to be executed, supports undo functionality, and can be used to implement logging and transaction mechanisms.

Template Method Design Pattern

Въведение

Патернът Template Method дефинира скелет на алгоритъм в метод, като делегира някои стъпки на подкласовете. Той позволява на подкласовете да преопределят определени стъпки на алгоритъма, без да променят неговата структура.

Пример

Ето един пример за имплементация на Template Method в C++:

#include <iostream>
class AbstractClass {
public:
void TemplateMethod() {
BaseOperation1();
RequiredOperation1();
BaseOperation2();
RequiredOperation2();
Hook1();
BaseOperation3();
Hook2();
}
protected:
void BaseOperation1() const {
std::cout << "AbstractClass: BaseOperation1\n";
}
void BaseOperation2() const {
std::cout << "AbstractClass: BaseOperation2\n";
}
void BaseOperation3() const {
std::cout << "AbstractClass: BaseOperation3\n";
}
virtual void RequiredOperation1() const = 0;
virtual void RequiredOperation2() const = 0;
virtual void Hook1() const {}
virtual void Hook2() const {}
};
class ConcreteClass : public AbstractClass {
protected:
void RequiredOperation1() const override {
std::cout << "ConcreteClass: RequiredOperation1\n";
}
void RequiredOperation2() const override {
std::cout << "ConcreteClass: RequiredOperation2\n";
}
void Hook1() const override {
std::cout << "ConcreteClass: Hook1\n";
}
};
int main() {
ConcreteClass concrete;
concrete.TemplateMethod();
return 0;
}

В този пример, AbstractClass дефинира скелет на алгоритъм в метода TemplateMethod(). Подкласът ConcreteClass преопределя определени стъпки на алгоритъма, като оставя други да бъдат изпълнени от базовия клас.

Този код ще изведе: AbstractClass: BaseOperation1 ConcreteClass: RequiredOperation1 AbstractClass: BaseOperation2 ConcreteClass: RequiredOperation2 ConcreteClass: Hook1 AbstractClass: BaseOperation3

Приложимост

Патернът Template Method е подходящ за използване в следните случаи:

  • Когато имате няколко класа, които изпълняват сходни алгоритми с малки разлики. Вместо да дублирате кода, можете да дефинирате общия алгоритъм в базов клас и да позволите на подкласовете да преопределят специфичните стъпки.
  • Когато искате да контролирате разширяването на алгоритъма. Чрез дефиниране на скелета на алгоритъма в базовия клас, можете да гарантирате, че структурата на алгоритъма остава непроменена, докато позволявате на подкласовете да добавят или променят определени стъпки.
  • Когато искате да избегнете дублиране на код. Патернът Template Method позволява повторно използване на общия код в базовия клас, като същевременно предоставя гъвкавост на подкласовете да добавят специфично поведение.

Примери за използване на Template Method включват алгоритми за сортиране, обработка на данни и игри, където основната логика е една и съща, но специфичните стъпки могат да варират.

Последици

Патернът Template Method има следните последици:

  • Повторно използване на код: Общият код на алгоритъма се дефинира в базовия клас, което позволява на подкласовете да използват този код без да го дублират.
  • Лесно разширяване: Подкласовете могат да преопределят само специфичните стъпки на алгоритъма, което улеснява разширяването и модифицирането на поведението.
  • Контрол върху алгоритъма: Базовият клас контролира структурата на алгоритъма, което гарантира, че основната логика остава непроменена.
  • Сложност: Добавянето на нови подкласове може да увеличи сложността на системата, особено ако има много различни вариации на алгоритъма.
  • Зависимост от базовия клас: Подкласовете са силно зависими от базовия клас, което може да затрудни поддръжката и разширяването на кода в бъдеще.

Участници

  • AbstractClass: Дефинира скелета на алгоритъма и съдържа методите, които ще бъдат изпълнени от подкласовете.
  • ConcreteClass: Преопределя специфичните стъпки на алгоритъма, като наследява от AbstractClass.

Сътрудничество

ConcreteClass наследява AbstractClass и преопределя специфичните стъпки на алгоритъма. AbstractClass дефинира скелета на алгоритъма и съдържа методите, които ще бъдат изпълнени от подкласовете. ConcreteClass използва тези методи, за да изпълни алгоритъма със своите специфични стъпки.

Примери

#include <iostream>
class Game {
public:
void play() {
start();
while (!isOver()) {
makeMove();
}
end();
}
protected:
virtual void start() = 0;
virtual bool isOver() = 0;
virtual void makeMove() = 0;
virtual void end() = 0;
};
class Chess : public Game {
protected:
void start() override {
std::cout << "Starting a game of chess.\n";
}
bool isOver() override {
static int moves = 0;
return ++moves == 10;
}
void makeMove() override {
std::cout << "Making a move in chess.\n";
}
void end() override {
std::cout << "Ending the game of chess.\n";
}
};
int main() {
Chess chess;
chess.play();
return 0;
}

В този пример, класът Game дефинира скелета на алгоритъма за игра в метода play(). Подкласът Chess преопределя специфичните стъпки на алгоритъма, като оставя други да бъдат изпълнени от базовия клас.

Този код ще изведе: Starting a game of chess. Making a move in chess. Making a move in chess. Making a move in chess. Making a move in chess. Making a move in chess. Making a move in chess. Making a move in chess. Making a move in chess. Making a move in chess. Ending the game of chess.

Вариант

Друг вариант на Template Method патерна е използването на шаблони (templates) в C++. Това позволява дефинирането на алгоритъма в базовия клас, като същевременно предоставя гъвкавост на подкласовете да преопределят специфичните стъпки чрез шаблонни параметри.

#include <iostream>
template <typename T>
class TemplateMethod {
public:
void execute() {
static_cast<T*>(this)->step1();
static_cast<T*>(this)->step2();
static_cast<T*>(this)->step3();
}
};
class ConcreteImplementation : public TemplateMethod<ConcreteImplementation> {
public:
void step1() {
std::cout << "ConcreteImplementation: Step 1\n";
}
void step2() {
std::cout << "ConcreteImplementation: Step 2\n";
}
void step3() {
std::cout << "ConcreteImplementation: Step 3\n";
}
};
int main() {
ConcreteImplementation impl;
impl.execute();
return 0;
}

В този пример, TemplateMethod е шаблонен клас, който дефинира алгоритъма в метода execute(). Класът ConcreteImplementation наследява TemplateMethod и преопределя специфичните стъпки на алгоритъма чрез шаблонни параметри.

Този код ще изведе: ConcreteImplementation: Step 1 ConcreteImplementation: Step 2 ConcreteImplementation: Step 3

Известни употреби

  • Алгоритми за сортиране, като например QuickSort и MergeSort, където основната логика на сортиране е една и съща, но специфичните стъпки могат да варират.
  • Обработка на данни, където различни формати на данни могат да бъдат обработени чрез различни стъпки, но основната логика за обработка остава същата.
  • Игри, където основната логика на играта е една и съща, но специфичните стъпки за различни игри могат да варират.

Реализация

  • Дефинирайте абстрактен базов клас: Създайте абстрактен базов клас, който съдържа метода Template Method и дефинира скелета на алгоритъма.
  • Декларирайте абстрактни методи: Декларирайте абстрактни методи в базовия клас за стъпките на алгоритъма, които трябва да бъдат преопределени от подкласовете.
  • Имплементирайте базови методи: Имплементирайте базовите методи в базовия клас, които ще бъдат използвани от Template Method.
  • Създайте подкласове: Създайте подкласове, които наследяват базовия клас и преопределят абстрактните методи със специфични реализации.
  • Използвайте Template Method: Използвайте Template Method в подкласовете, за да изпълните алгоритъма със специфичните стъпки.

Ползи

Патернът Template Method предлага следните ползи:

  • Повторно използване на код: Позволява използването на общ код в базовия клас, като същевременно предоставя гъвкавост на подкласовете да добавят специфично поведение.
  • Лесно разширяване: Улеснява разширяването и модифицирането на поведението чрез преопределяне на специфичните стъпки на алгоритъма.
  • Контрол върху алгоритъма: Гарантира, че основната логика на алгоритъма остава непроменена, докато позволява на подкласовете да добавят или променят определени стъпки.

Недостатъци

  • Сложност: Добавянето на нови подкласове може да увеличи сложността на системата, особено ако има много различни вариации на алгоритъма.
  • Зависимост от базовия клас: Подкласовете са силно зависими от базовия клас, което може да затрудни поддръжката и разширяването на кода в бъдеще.

Варианти

  • Шаблонен метод с шаблони (Template Method with Templates): Използването на шаблони в C++ позволява дефинирането на алгоритъма в базовия клас, като същевременно предоставя гъвкавост на подкласовете да преопределят специфичните стъпки чрез шаблонни параметри. Пример за това беше показан по-горе в секцията "Вариант".
  • Шаблонен метод с делегиране (Template Method with Delegation): Вместо да използвате наследяване, можете да използвате делегиране, за да дефинирате стъпките на алгоритъма. Това може да намали зависимостта от базовия клас и да улесни разширяването на кода.
  • Шаблонен метод с функционални обекти (Template Method with Functors): Използването на функционални обекти или ламбда функции за дефиниране на стъпките на алгоритъма може да предостави по-голяма гъвкавост и да улесни промяната на поведението по време на изпълнение.

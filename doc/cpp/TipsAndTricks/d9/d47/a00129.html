Prototype Design Pattern

Въведение

Патернът Prototype позволява създаването на нови обекти чрез клониране на съществуващи обекти, вместо да се създават нови инстанции от класове. Това е полезно, когато създаването на нови обекти е скъпо или сложно.

В този пример, класът Prototype дефинира интерфейс за клониране на себе си. Класът ConcretePrototype имплементира този интерфейс и позволява създаването на нови обекти чрез клониране.

#include <iostream>
#include <string>
#include <memory>
class Prototype {
public:
virtual ~Prototype() {}
virtual std::unique_ptr<Prototype> clone() const = 0;
virtual void print() const = 0;
};
class ConcretePrototype : public Prototype {
private:
std::string name;
public:
ConcretePrototype(const std::string &name) : name(name) {}
std::unique_ptr<Prototype> clone() const override {
return std::make_unique<ConcretePrototype>(*this);
}
void print() const override {
std::cout << "ConcretePrototype: " << name << std::endl;
}
};
int main() {
std::unique_ptr<Prototype> prototype = std::make_unique<ConcretePrototype>("Prototype 1");
std::unique_ptr<Prototype> clone = prototype->clone();
prototype->print();
clone->print();
return 0;
}

В този пример, класът ConcretePrototype имплементира интерфейса Prototype и позволява създаването на нови обекти чрез клониране на себе си. Класът ConcretePrototype има член-данна name, която се клонира при създаване на нови обекти.

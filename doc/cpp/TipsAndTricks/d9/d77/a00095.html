Strategy Design Pattern

Въведение

Патернът Strategy позволява на алгоритмите да се избират по време на изпълнение. Това е полезно, когато имаме множество алгоритми за изпълнение на дадена задача и искаме да изберем един от тях динамично.

#include <iostream>
#include <memory>
// Интерфейс за стратегията
class Strategy {
public:
virtual ~Strategy() = default;
virtual void execute() const = 0;
};
// Конкретна стратегия A
class ConcreteStrategyA : public Strategy {
public:
void execute() const override {
std::cout << "Изпълняване на стратегия A\n";
}
};
// Конкретна стратегия B
class ConcreteStrategyB : public Strategy {
public:
void execute() const override {
std::cout << "Изпълняване на стратегия B\n";
}
};
// Контекст, който използва стратегията
class Context {
private:
std::unique_ptr<Strategy> strategy;
public:
Context(std::unique_ptr<Strategy> strategy) : strategy(std::move(strategy)) {}
void setStrategy(std::unique_ptr<Strategy> newStrategy) {
strategy = std::move(newStrategy);
}
void executeStrategy() const {
strategy->execute();
}
};
int main() {
Context context(std::make_unique<ConcreteStrategyA>());
context.executeStrategy();
context.setStrategy(std::make_unique<ConcreteStrategyB>());
context.executeStrategy();
return 0;
}

В този пример, контекстът използва различни стратегии за изпълнение на задачата. Класовете ConcreteStrategyA и ConcreteStrategyB имплементират различни алгоритми, които могат да бъдат избрани динамично по време на изпълнение.

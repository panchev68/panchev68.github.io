Command Design Pattern

Въведение

Патернът Command е поведенчески дизайн патерн, който превръща заявките в обекти, позволявайки по този начин параметризация на клиентите с различни заявки, опашка или логиране на заявки, и поддръжка на операции за отмяна.

Пример

Пример за използване на патерна Command:

#include <iostream>
#include <vector>
#include <memory>
// Интерфейс за командите
class Command {
public:
virtual ~Command() {}
virtual void execute() const = 0;
};
// Конкретна команда
class ConcreteCommand : public Command {
private:
std::string payload_;
public:
explicit ConcreteCommand(std::string payload) : payload_(std::move(payload)) {}
void execute() const override {
std::cout << "ConcreteCommand: " << payload_ << std::endl;
}
};
// Инициаторът
class Invoker {
private:
std::vector<std::unique_ptr<Command>> history_;
public:
void storeAndExecute(std::unique_ptr<Command> command) {
command->execute();
history_.push_back(std::move(command));
}
};
int main() {
Invoker invoker;
invoker.storeAndExecute(std::make_unique<ConcreteCommand>("Action 1"));
invoker.storeAndExecute(std::make_unique<ConcreteCommand>("Action 2"));
return 0;
}

В този пример, класът Invoker съхранява и изпълнява команди, които са обекти, имплементиращи интерфейса Command. Класът ConcreteCommand представлява конкретна команда, която изпълнява определено действие.

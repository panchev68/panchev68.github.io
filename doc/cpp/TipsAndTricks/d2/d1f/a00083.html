Facade Design Pattern

Въведение

Патернът Facade предоставя унифициран интерфейс към набор от интерфейси в подсистема. Facade дефинира по-високо ниво на интерфейс, което улеснява използването на подсистемата.

Пример

Ето един пример за имплементация на патерна Facade в C++:

#include <iostream>
class Subsystem1 {
public:
void operation1() const {
std::cout << "Subsystem1: Ready!\n";
}
void operationN() const {
std::cout << "Subsystem1: Go!\n";
}
};
class Subsystem2 {
public:
void operation1() const {
std::cout << "Subsystem2: Get ready!\n";
}
void operationZ() const {
std::cout << "Subsystem2: Fire!\n";
}
};
class Facade {
protected:
Subsystem1 *subsystem1_;
Subsystem2 *subsystem2_;
public:
Facade(Subsystem1 *subsystem1 = nullptr, Subsystem2 *subsystem2 = nullptr) {
this->subsystem1_ = subsystem1 ?: new Subsystem1;
this->subsystem2_ = subsystem2 ?: new Subsystem2;
}
~Facade() {
delete subsystem1_;
delete subsystem2_;
}
void operation() {
subsystem1_->operation1();
subsystem1_->operationN();
subsystem2_->operation1();
subsystem2_->operationZ();
}
};
void clientCode(Facade *facade) {
facade->operation();
}
int main() {
Subsystem1 *subsystem1 = new Subsystem1;
Subsystem2 *subsystem2 = new Subsystem2;
Facade *facade = new Facade(subsystem1, subsystem2);
clientCode(facade);
delete facade;
return 0;
}

В този пример, класът Facade предоставя унифициран интерфейс към методите на класовете Subsystem1 и Subsystem2. Клиентският код използва Facade, за да работи с подсистемата, без да се налага да разбира сложността й.

Заключение

Патернът Facade е полезен, когато имаме сложна система с много взаимодействащи си класове и искаме да предоставим опростен интерфейс за работа с нея. Той помага за намаляване на зависимостите между клиентския код и сложната система, като същевременно подобрява четимостта и поддръжката на кода. Въпреки това, трябва да се внимава да не се създаде прекалено много логика във Facade класа, тъй като това може да доведе до нови проблеми със сложността.

Adapter Design Pattern

В тази страница ще разгледаме Adapter дизайн патърна в C++.

Какво е Adapter?

Adapter е дизайн патърн, който позволява обекти с несъвместими интерфейси да работят заедно. Той действа като мост между два несъвместими интерфейса, като преобразува интерфейса на един клас в интерфейс, който клиентът очаква.

Пример за Adapter:

#include <iostream>
using namespace std;
// Целеви интерфейс
class Target {
public:
virtual void request() {
cout << "Target request" << endl;
}
};
// Клас, който трябва да бъде адаптиран
class Adaptee {
public:
void specificRequest() {
cout << "Adaptee specific request" << endl;
}
};
// Адаптер, който прави Adaptee съвместим с Target
class Adapter : public Target {
public:
Adapter(Adaptee* adaptee) : adaptee(adaptee) {}
void request() override {
adaptee->specificRequest();
}
private:
Adaptee* adaptee;
};
int main() {
Adaptee* adaptee = new Adaptee();
Target* adapter = new Adapter(adaptee);
adapter->request(); // Output: Adaptee specific request
delete adapter;
delete adaptee;
return 0;
}

В този пример, класът Adaptee има метод specificRequest, който е несъвместим с интерфейса Target. Класът Adapter наследява Target и прави Adaptee съвместим с Target, като преобразува извикванията на метода request към метода specificRequest.

Заключение:

Adapter е полезен дизайн патърн, когато искаме да направим два несъвместими интерфейса съвместими. Той предоставя гъвкавост и разширяемост при интегриране на съществуващи класове с нови системи.

Decorator Design Pattern

В тази страница ще разгледаме Decorator дизайн патърна в C++.

Какво е Decorator?

Decorator е дизайн патърн, който позволява динамично добавяне на нови функционалности към обекти, без да се променя тяхната структура. Той използва композиция вместо наследяване, за да разшири функционалността на обектите.

Пример за Decorator:

#include <iostream>
using namespace std;
// Интерфейс за компонентите
class Component {
public:
virtual void operation() = 0;
};
// Конкретен компонент
class ConcreteComponent : public Component {
public:
void operation() override {
cout << "ConcreteComponent operation" << endl;
}
};
// Базов клас за декораторите
class Decorator : public Component {
public:
Decorator(Component* component) : component(component) {}
void operation() override {
if (component) {
component->operation();
}
}
protected:
Component* component;
};
// Конкретен декоратор A
class ConcreteDecoratorA : public Decorator {
public:
ConcreteDecoratorA(Component* component) : Decorator(component) {}
void operation() override {
Decorator::operation();
cout << "ConcreteDecoratorA operation" << endl;
}
};
// Конкретен декоратор B
class ConcreteDecoratorB : public Decorator {
public:
ConcreteDecoratorB(Component* component) : Decorator(component) {}
void operation() override {
Decorator::operation();
cout << "ConcreteDecoratorB operation" << endl;
}
};
int main() {
Component* component = new ConcreteComponent();
Component* decoratorA = new ConcreteDecoratorA(component);
Component* decoratorB = new ConcreteDecoratorB(decoratorA);
decoratorB->operation();
// Output:
// ConcreteComponent operation
// ConcreteDecoratorA operation
// ConcreteDecoratorB operation
delete decoratorB;
delete decoratorA;
delete component;
return 0;
}

В този пример, класът ConcreteComponent имплементира интерфейса Component. Класовете ConcreteDecoratorA и ConcreteDecoratorB разширяват функционалността на ConcreteComponent чрез композиция.

Заключение:

Decorator е полезен дизайн патърн, когато искаме динамично да добавяме нови функционалности към обекти, без да променяме тяхната структура. Той предоставя гъвкавост и разширяемост при разширяване на функционалността на обектите.

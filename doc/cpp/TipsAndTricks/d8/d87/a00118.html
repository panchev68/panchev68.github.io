Command Design Pattern

Въведение

Патернът Command е поведенчески дизайн патерн, който инкапсулира заявките като обекти, позволявайки параметризация на клиентите с различни заявки, опашка или логиране на заявки, както и поддръжка на операции за отмяна. Той разделя обекта, който инициира операцията, от този, който я изпълнява, чрез дефиниране на абстрактен интерфейс за командите и конкретни реализации за всяко действие. Командите могат да се съхраняват, предават, логват и отменят, което улеснява реализирането на undo/redo, отложено изпълнение и проследяване на историята на действията. Този подход подобрява разширяемостта и поддръжката на кода, като позволява добавяне на нови команди без промяна на съществуващата логика.

Пример

Пример за използване на патерна Command:

/**
* @file Command.hpp
* Defines the Command design pattern implementation for encapsulating requests as objects, enabling parameterization,
* queuing, logging, and undo operations. This file provides the Command interface, a concrete command implementation,
* and an Invoker class that manages command execution and history. The example demonstrates how to use the pattern
* for flexible and extensible command processing in C++ applications.
*/
#include <iostream>
#include <memory>
#include <string>
#include <vector>
/**
* Command interface for encapsulating a request as an object.
* This abstract class defines the interface for executing commands, enabling parameterization and queuing of requests.
*/
class Command {
public:
/**
* Destructor for Command interface.
*/
virtual ~Command() = default;
/**
* Executes the encapsulated command action.
*/
virtual void execute() const = 0;
};
/**
* Concrete command that implements a specific action.
* This class stores a payload and implements the execute method to perform a concrete operation using the payload.
*/
class ConcreteCommand : public Command {
private:
std::string payload; ///< Payload for the command.
public:
/**
* Constructs a ConcreteCommand with the specified payload.
* @param payload The string payload to be used by the command.
*/
explicit ConcreteCommand(std::string payload) : payload(std::move(payload)) {}
/**
* Executes the concrete command action by printing the payload.
*/
void execute() const override { std::cout << "ConcreteCommand: " << payload << std::endl; }
};
/**
* Invoker class that stores and executes commands.
* This class manages the execution and history of commands, allowing for flexible command processing and undo support.
*/
class Invoker {
private:
std::vector<std::unique_ptr<Command>> history; ///< History of executed commands.
public:
/**
* Executes the given command and stores it in the history.
* @param command[out] The unique pointer to the command to execute and store.
*/
void storeAndExecute(std::unique_ptr<Command> command) {
command->execute();
history.push_back(std::move(command));
}
};
/**
* Entry point for demonstrating the Command pattern.
* @return Returns 0 upon successful execution.
*/
int main() {
Invoker invoker;
invoker.storeAndExecute(std::make_unique<ConcreteCommand>("Action 1"));
invoker.storeAndExecute(std::make_unique<ConcreteCommand>("Action 2"));
return 0;
}

В този пример класът Invoker съхранява и изпълнява команди, които са обекти, имплементиращи интерфейса Command. Класът ConcreteCommand представлява конкретна команда, която изпълнява определено действие.

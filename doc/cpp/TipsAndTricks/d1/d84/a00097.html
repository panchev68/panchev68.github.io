Adapter Pattern

Въведение

Патернът Adapter позволява обекти с несъвместими интерфейси да работят заедно. Той действа като мост между два несъвместими интерфейса, като преобразува интерфейса на един клас в интерфейс, който клиентите очакват. Това позволява на класовете с различни интерфейси да работят заедно, без да се променят.

Пример

Ето един пример за имплементация на патерна Adapter в C++:

/**
* @file Adapter.hpp
* Demonstrates the Adapter design pattern in C++ for bridging incompatible interfaces.
* This file provides a sample implementation of the Adapter pattern, allowing objects with incompatible
* interfaces to collaborate by introducing an adapter class that translates requests from the expected
* interface to the adaptee's interface. The example showcases how to use inheritance and composition to
* achieve interface compatibility without modifying existing code, which is particularly useful in embedded
* systems and legacy integration scenarios.
*/
#pragma once
#include <iostream>
/**
* Represents the target interface expected by clients.
* This class defines the interface that client code expects to use, enabling polymorphic behavior and
* allowing adapters to provide compatibility with other interfaces by inheriting from this class.
*/
class ClientInterface {
public:
/**
* Handles the default request using the client interface.
* @return None.
*/
virtual void handleRequest() const { std::cout << "ClientInterface: The default behavior." << std::endl; }
};
/**
* Represents an existing class with an incompatible interface.
* This class provides a specific interface that is not compatible with the client interface, requiring an
* adapter to enable interoperability with client code expecting the client interface.
*/
class LegacyService {
public:
/**
* Handles a specific request using the legacy service's interface.
* @return None.
*/
void performLegacyOperation() const { std::cout << "LegacyService: Legacy operation." << std::endl; }
};
/**
* Adapts the LegacyService interface to the ClientInterface.
* This class implements the client interface and internally uses a legacy service instance to translate
* requests, enabling client code to interact with legacy service objects as if they were client interfaces
* without modifying either the client or legacy service code.
*/
class LegacyAdapter : public ClientInterface {
private:
LegacyService * legacyService_; ///< Pointer to the legacy service instance used for request translation.
public:
/**
* Constructs a LegacyAdapter with the given legacy service instance.
* @param legacyService Pointer to the legacy service object to be adapted.
*/
LegacyAdapter(LegacyService * legacyService) : legacyService_(legacyService) {}
/**
* Handles the request by translating it to the legacy service's operation.
* @return None.
* @see LegacyService::performLegacyOperation
*/
void handleRequest() const override { legacyService_->performLegacyOperation(); }
};
/**
* Entry point demonstrating the Adapter pattern usage.
* @return Exit status code.
*/
int main() {
LegacyService * legacyService = new LegacyService;
ClientInterface * client = new LegacyAdapter(legacyService);
client->handleRequest();
delete legacyService;
delete client;
return 0;
}

В този пример, класът Target дефинира интерфейс, който клиентите очакват. Класът Adaptee има интерфейс, който не е съвместим с интерфейса на Target. Класът Adapter преобразува интерфейса на Adaptee в интерфейса на Target, така че клиентите да могат да използват Adaptee като Target.

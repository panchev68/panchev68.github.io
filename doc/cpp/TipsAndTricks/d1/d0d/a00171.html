PIMPL idiom in C++

PIMPL (Pointer to IMPLementation) идиомът е техника за скриване на детайлите на реализацията на клас в C++.

PIMPL идиомът се използва за намаляване на зависимостите между класовете и за подобряване на времето за компилиране. Той също така помага за запазване на бинарната съвместимост при промени в реализацията на класа.

Пример за PIMPL идиом:

// MyClass.h
class MyClass {
public:
MyClass();
~MyClass();
void doSomething();
private:
class Impl;
Impl* pImpl;
};
// MyClass.cpp
#include "MyClass.h"
class MyClass::Impl {
public:
void doSomethingImpl() {
// Реализация на метода
}
};
MyClass::MyClass() : pImpl(new Impl()) {}
MyClass::~MyClass() {
delete pImpl;
}
void MyClass::doSomething() {
pImpl->doSomethingImpl();
}

В този пример, детайлите на реализацията на класа MyClass са скрити в класа Impl, който е дефиниран в cpp файла. Това позволява промени в реализацията на класа без да се налага прекомпилиране на кода, който използва MyClass.

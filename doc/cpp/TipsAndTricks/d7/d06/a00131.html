Singleton Design Pattern

Въведение

Патернът Singleton гарантира, че даден клас има само една инстанция и предоставя глобална точка за достъп до нея. Това е полезно, когато е необходимо да се контролира достъпа до някакъв ресурс, като например база данни или конфигурационен файл.

#include <iostream>
#include <mutex>
class Singleton {
private:
static Singleton* instance;
static std::mutex mutex;
// Приватен конструктор за предотвратяване на създаване на обекти
Singleton() {}
public:
// Забраняване на копиране и присвояване
Singleton(Singleton &other) = delete;
void operator=(const Singleton &) = delete;
static Singleton* getInstance() {
std::lock_guard<std::mutex> lock(mutex);
if (instance == nullptr) {
instance = new Singleton();
}
return instance;
}
void someBusinessLogic() {
std::cout << "Executing some business logic." << std::endl;
}
};
// Инициализация на статичните членове
Singleton* Singleton::instance = nullptr;
std::mutex Singleton::mutex;
int main() {
Singleton* singleton = Singleton::getInstance();
singleton->someBusinessLogic();
return 0;
}

В този пример, класът Singleton гарантира, че има само една инстанция и предоставя метод за достъп до нея. Използва се mutex за синхронизация на достъпа в многопоточна среда.

Bridge Design Pattern

Патернът Bridge разделя абстракцията от нейната имплементация, така че двете могат да се променят независимо.

Това позволява по-голяма гъвкавост и разширяемост на кода.

Пример

В този пример класът Abstraction дефинира интерфейса, а класът RefinedAbstraction го разширява. Имплементацията е представена от интерфейса Implementor и неговите конкретни реализации ConcreteImplementorA и ConcreteImplementorB. Това позволява абстракцията и имплементацията да се развиват независимо, като се улеснява разширяването и поддръжката на системата.

/**
* @file Bridge.hpp
* Implementation of the Bridge design pattern, separating abstraction from implementation to allow both to
* vary independently. This file defines the abstraction and implementor interfaces, their concrete
* implementations, and demonstrates usage in a simple main function. The design enables flexible extension
* and decoupling of abstraction and implementation, supporting scalable and maintainable code structures.
*/
#pragma once
#include <cstdint>
#include <memory>
/**
* Interface for implementation classes in the Bridge pattern.
* Provides an abstract operation to be implemented by concrete implementors, enabling decoupling from abstraction.
*/
class Implementor {
public:
/**
* Performs the implementation-specific operation.
* @pre Must be implemented by derived classes.
*/
virtual void operation() = 0;
/**
* Destroys the Implementor instance.
*/
virtual ~Implementor() = default;
};
/**
* Concrete implementor A for the Bridge pattern.
* Implements the operation method with a specific behavior for variant A.
*/
class ConcreteImplementorA : public Implementor {
public:
/**
* Performs the operation specific to ConcreteImplementorA.
* @pre Must be called on a valid ConcreteImplementorA instance.
*/
void operation() override {
// Implementation A
}
};
/**
* Concrete implementor B for the Bridge pattern.
* Implements the operation method with a specific behavior for variant B.
*/
class ConcreteImplementorB : public Implementor {
public:
/**
* Performs the operation specific to ConcreteImplementorB.
* @pre Must be called on a valid ConcreteImplementorB instance.
*/
void operation() override {
// Implementation B
}
};
/**
* Abstraction class in the Bridge pattern.
* Maintains a reference to an Implementor and delegates operations to it, enabling flexible implementation changes.
*/
class Abstraction {
protected:
std::unique_ptr<Implementor> implementor; ///< Owning pointer to the implementation object
public:
/**
* Constructs an Abstraction with the given implementor.
* @param impl Unique pointer to the Implementor instance.
* @pre impl must not be nullptr.
*/
explicit Abstraction(std::unique_ptr<Implementor> impl) : implementor(std::move(impl)) {}
/**
* Delegates the operation to the implementor.
* @pre implementor must not be nullptr.
*/
virtual void operation() { implementor->operation(); }
/**
* Destroys the Abstraction instance.
*/
virtual ~Abstraction() = default;
};
/**
* Refined abstraction in the Bridge pattern.
* Extends Abstraction to provide additional or specialized behavior while still delegating to the implementor.
*/
class RefinedAbstraction : public Abstraction {
public:
/**
* Constructs a RefinedAbstraction with the given implementor.
* @param impl Unique pointer to the Implementor instance.
* @pre impl must not be nullptr.
*/
explicit RefinedAbstraction(std::unique_ptr<Implementor> impl) : Abstraction(std::move(impl)) {}
/**
* Performs an extended operation and delegates to the implementor.
* @pre implementor must not be nullptr.
*/
void operation() override {
// Extended operation
implementor->operation();
}
};
int main() {
/**
* Demonstrates usage of the Bridge pattern with different implementors.
* @return Exit status code.
* @pre None.
*/
auto absA = std::make_unique<RefinedAbstraction>(std::make_unique<ConcreteImplementorA>());
absA->operation();
auto absB = std::make_unique<RefinedAbstraction>(std::make_unique<ConcreteImplementorB>());
absB->operation();
return 0;
}

В този пример абстракцията и имплементацията могат да се променят независимо една от друга, което прави кода по-гъвкав и лесен за поддръжка.

Заключение

Патернът Bridge разделя абстракцията от нейната имплементация, така че двете могат да се променят независимо. Това позволява по-голяма гъвкавост и разширяемост на кода.

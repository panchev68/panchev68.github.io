Decorator Design Pattern документация

Въведение

Патернът Decorator позволява динамично добавяне на поведение към обекти, без да се променя техният код. Това се постига чрез създаване на класове, които "опаковат" оригиналните обекти и добавят ново поведение.

Пример

Ето един пример за имплементация на патерна Decorator в C++:

#include <iostream>
#include <memory>
// Базов клас Component
class Component {
public:
virtual ~Component() {}
virtual void operation() const = 0;
};
// Конкретен компонент
class ConcreteComponent : public Component {
public:
void operation() const override {
std::cout << "ConcreteComponent operation\n";
}
};
// Базов клас Decorator
class Decorator : public Component {
protected:
std::unique_ptr<Component> component_;
public:
Decorator(std::unique_ptr<Component> component) : component_(std::move(component)) {}
void operation() const override {
component_->operation();
}
};
// Конкретен декоратор A
class ConcreteDecoratorA : public Decorator {
public:
ConcreteDecoratorA(std::unique_ptr<Component> component) : Decorator(std::move(component)) {}
void operation() const override {
Decorator::operation();
std::cout << "ConcreteDecoratorA operation\n";
}
};
// Конкретен декоратор B
class ConcreteDecoratorB : public Decorator {
public:
ConcreteDecoratorB(std::unique_ptr<Component> component) : Decorator(std::move(component)) {}
void operation() const override {
Decorator::operation();
std::cout << "ConcreteDecoratorB operation\n";
}
};
int main() {
std::unique_ptr<Component> component = std::make_unique<ConcreteComponent>();
component = std::make_unique<ConcreteDecoratorA>(std::move(component));
component = std::make_unique<ConcreteDecoratorB>(std::move(component));
component->operation();
return 0;
}

В този пример, класовете ConcreteDecoratorA и ConcreteDecoratorB добавят ново поведение към ConcreteComponent, без да променят неговия код.

Заключение

Патернът Decorator е мощен инструмент за добавяне на функционалност към обекти по динамичен начин. Той позволява гъвкавост и повторна употреба на кода, като същевременно спазва принципа за отвореност-затвореност (Open/Closed Principle). Въпреки това, прекомерното използване на декоратори може да доведе до сложност и трудност при проследяване на кода. Затова е важно да се използва разумно и само когато е необходимо.

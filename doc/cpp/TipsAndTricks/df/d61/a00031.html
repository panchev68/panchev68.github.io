Strategy Design Pattern

В тази страница ще разгледаме Strategy дизайн патърна в C++.

Какво е Strategy?

Strategy е дизайн патърн, който дефинира семейство от алгоритми, капсулира всеки от тях и ги прави взаимозаменяеми. Той позволява на клиента да избира алгоритъм по време на изпълнение.

Пример за Strategy:

#include <iostream>
using namespace std;
// Интерфейс за стратегията
class Strategy {
public:
virtual void execute() = 0;
};
// Конкретна стратегия A
class ConcreteStrategyA : public Strategy {
public:
void execute() override {
cout << "Executing Strategy A" << endl;
}
};
// Конкретна стратегия B
class ConcreteStrategyB : public Strategy {
public:
void execute() override {
cout << "Executing Strategy B" << endl;
}
};
// Контекст, който използва стратегията
class Context {
public:
void setStrategy(Strategy* strategy) {
this->strategy = strategy;
}
void executeStrategy() {
if (strategy) {
strategy->execute();
}
}
private:
Strategy* strategy = nullptr;
};
int main() {
Context context;
ConcreteStrategyA strategyA;
ConcreteStrategyB strategyB;
context.setStrategy(&strategyA);
context.executeStrategy(); // Output: Executing Strategy A
context.setStrategy(&strategyB);
context.executeStrategy(); // Output: Executing Strategy B
return 0;
}

В този пример, класът Context използва интерфейса Strategy за изпълнение на алгоритъм. Конкретните стратегии ConcreteStrategyA и ConcreteStrategyB имплементират този интерфейс и предоставят различни реализации на алгоритъма.

Заключение:

Strategy е полезен дизайн патърн, когато искаме да дефинираме семейство от алгоритми и да ги направим взаимозаменяеми. Той предоставя гъвкавост и разширяемост при избора на алгоритъм по време на изпълнение.

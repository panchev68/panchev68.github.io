Memento Design Pattern

Въведение

Патернът Memento позволява запазването и възстановяването на състоянието на обект, без да се нарушава неговата инкапсулация. Това е полезно за реализиране на undo/redo функционалност или за съхраняване на състоянието на обект в определен момент.

#include <iostream>
#include <string>
#include <vector>
class Memento {
public:
Memento(const std::string &state) : state_(state) {}
std::string getState() const {
return state_;
}
private:
std::string state_;
};
class Originator {
public:
void setState(const std::string &state) {
std::cout << "Originator: Setting state to " << state << "\n";
state_ = state;
}
std::string getState() const {
return state_;
}
Memento save() const {
return Memento(state_);
}
void restore(const Memento &memento) {
state_ = memento.getState();
std::cout << "Originator: State after restoring from Memento: " << state_ << "\n";
}
private:
std::string state_;
};
class Caretaker {
public:
void backup() {
std::cout << "Caretaker: Saving Originator's state...\n";
mementos_.push_back(originator_.save());
}
void undo() {
if (!mementos_.empty()) {
Memento memento = mementos_.back();
mementos_.pop_back();
std::cout << "Caretaker: Restoring state to " << memento.getState() << "\n";
originator_.restore(memento);
} else {
std::cout << "Caretaker: No mementos to restore.\n";
}
}
void setOriginator(const Originator &originator) {
originator_ = originator;
}
private:
std::vector<Memento> mementos_;
Originator originator_;
};
int main() {
Originator originator;
Caretaker caretaker;
caretaker.setOriginator(originator);
originator.setState("State1");
caretaker.backup();
originator.setState("State2");
caretaker.backup();
originator.setState("State3");
caretaker.undo();
originator.setState("State4");
caretaker.undo();
return 0;
}

В този пример, Originator класът създава Memento обекти, които съхраняват неговото състояние. Caretaker класът управлява тези Memento обекти и може да възстановява състоянието на Originator при необходимост.

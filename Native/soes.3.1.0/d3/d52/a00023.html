Detailed Description

Definitions for EtherCAT Slave Controller (ESC) CoE (CANopen over EtherCAT) object dictionary.

This header file contains macros for object types, data types, and access types used in the ESC CoE object dictionary.

Processes the CoE (CANopen over EtherCAT) protocol.

This function handles the processing of the CoE protocol, which is used for communication in EtherCAT networks. It manages the reception and transmission of CoE messages, ensuring proper communication between EtherCAT devices.

◆ sizeOfPDO()

uint16_t sizeOfPDO ( uint16_t index,
int * nmappings,
_SMmap * mappings,
int max_mappings )

Calculates the size of the Process Data Object (PDO) for a given index.

Parameters
indexThe index of the PDO to calculate the size for.
nmappingsPointer to an integer where the number of mappings will be stored.
smPointer to an array of _SMmap structures representing the Sync Manager mappings.
max_mappingsThe maximum number of mappings that can be processed.
Returns
The size of the PDO in bytes.

Calculate the size in Bytes of RxPDO or TxPDOs by adding the objects in SyncManager SDO 1C1x.

A list of mapped objects is created for fast lookup of dynamically mapped process data. The max size of the list (max_mappings) can be set to 0 if dynamic processdata is not supported.

The output variable nmappings is set to 0 if dynamic processdata is not supported. It is set to the number of mapped objects if dynamic processdata is supported, or -1 if the mapping was incorrect.

Parameters
[in]index= SM index
[out]nmappings= number of mapped objects in SM, or -1 if mapping is invalid
[out]mappings= list of mapped objects in SM
[out]max_mappings= max number of mapped objects in SM
Returns
size of RxPDO or TxPDOs in Bytes.
